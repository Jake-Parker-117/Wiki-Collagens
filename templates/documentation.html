What is Wiki Collagens?

Wiki Collagens is a free and open source webapp which can analyse COL domains in given fasta formatted protein sequences.

Tool inputs and outputs:


Wiki Collagens takes .fasta, .fa, .faa, and .txt files. The parameters of the tool which can be altered include the minimum length for a COL domain in triplets (default=10) and the maximum length of a GNG interruption (default=12, ie. G3-12G).
This tool provides 3 outputs: a flattened fasta .txt file of the COL domain containing proteins, an html file with the same protein sequences and highlighted COL domains and interuptions and finally a .txt file containing statistics about interuption and GXY prevalence.

Highlighted output explanation:


Non COL regions are not highlighted, COL domains are highlighted in yellow, GXG interruptions are highlighted in green, GNG interruptions are highlighted in cyan and GGG repeats are coloured red.

Example highlighted sequence: DSRSTEKTTGPTGLKGESGTATSTGTSGPAGLPGYGVKGHQGGGGGG

Collagen detection method and disclaimer:


The tool analyses sequences based on the amino acid sequence and does not take homology or probable helix formation into account. Rather it uses regular expressions to identify the repeating GXY amino acid patterns within COL domains. Therefore cases such as short repeating COL domains interspersed with short interuptons are highlighted as such rather than continual COL domains due to the sequence based nature of the tool and the ambiguity of such cases.

Disclaimers:

  1. This tool is not designed to find helix forming collagens, collagen-like proteins or true COL domains but rather to recognise COL domain patterns in amino acid sequeneces.
  2. Any gap characters (eg. -, * or ~) used to denote unknown amino acids will be removed from the sequences during processing.
  3. If fasta headers and sequences are not separated by a newline character (\n) the program will run on the header aswell as the sequence. It will also provide warnings in the output files for non-separated sequences, but will still provide a correct output.
