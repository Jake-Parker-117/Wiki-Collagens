What is Wiki Collagen?

Wiki Collagen is a free and open source webapp which can analyse collagen domains in given fasta formatted protein sequences.

Tool inputs and outputs:


Wiki Collagen takes .fasta, .fa, .faa, and .txt files. The parameters of the tool which can be altered include the minimum length for a collagen domain in triplets (default=10) and the maximum length of a GNG interruption (default=12, ie. G3-12G).
This tool provides 3 outputs: a flattened fasta .txt file of the collagen domain containing proteins, an html file with the same protein sequences and highlighted collagen domains and interruptions and finally a .txt file containing statistics about interruption and GXY prevalence.

Highlighted output explanation:


Non collagen triple helix regions are not highlighted, collagen domains are highlighted in yellow, GXG interruptions are highlighted in green, GNG interruptions are highlighted in cyan and GGG repeats are coloured red.

Example highlighted sequence: DSRSTEKTTGPTGLKGESGTATSTGTSGPAGLPGYGVKGHQGGGGGG

Collagen domain detection method and disclaimer:


The tool analyses sequences based on the amino acid sequence and does not take homology or probable helix formation into account. Rather it uses regular expressions to identify the repeating GXY amino acid patterns within collagen domains. Therefore cases such as short repeating collagen domains interspersed with short interruptons are highlighted as such rather than continual collagen domains due to the sequence based nature of the tool and the ambiguity of such cases.

Disclaimers:

  1. Privacy Notice: User information (eg. email, name, and private information) submitted via the contact form will be deleted within 1 month of submission. No user information is stored and all sequences are deleted after 1 hour. Please contact us if you have any questions.
  2. This tool is not designed to determine if the collagens, collagen-like proteins are helix forming or if they are true collagen domains but rather to recognise the collagen domain GXY pattern in amino acid sequeneces.
  3. Any gap characters (eg. -, * or ~) used to denote unknown amino acids will be removed from the sequences during processing.
  4. If fasta headers and sequences are not separated by a newline character (\n) the program will run on the header aswell as the sequence. It will also provide warnings in the output files for non-separated sequences, but will still provide a correct output.
